```python
# agent/followup.py — Seguimiento automatico post-consulta (FU-01)
# Programar, cancelar y procesar follow-ups de clientes que vieron propiedades

import os
import json
import logging
from datetime import datetime, timedelta
from sqlalchemy import select
from agent.memory import async_session, FollowUpSchedule

logger = logging.getLogger("agentkit")

FOLLOWUP_DELAY_HOURS = int(os.getenv("FOLLOWUP_DELAY_HOURS", "24"))

FOLLOWUP_MESSAGE = (
    "Hola! Soy Lucia de Bertero Inmobiliaria.\n\n"
    "Hace un tiempo te mostramos algunas propiedades. Pudiste verlas? Te intereso alguna?\n\n"
    "Si queres agendar una visita o tenes alguna consulta, estamos disponibles."
)
```

**Function 1: `programar_followup(telefono, propiedades)`**
- Upsert pattern (same as ConversationState in takeover.py): query for existing `pending` row with this telefono. If exists, update `scheduled_at = utcnow() + FOLLOWUP_DELAY_HOURS`, `propiedades_json = json.dumps(propiedades[:5])`, `updated_at = utcnow()`. If not exists, INSERT new row with status="pending".
- `propiedades` param is `list[dict]` — the property dicts shown to client.
- Use `datetime.utcnow()` for scheduled_at (UTC storage, consistent with existing models).

**Function 2: `cancelar_followup(telefono)`**
- Find all rows where telefono matches AND status=="pending". Set status="cancelled", updated_at=utcnow(). Commit.
- If no pending rows found, no-op (safe to call multiple times).

**Function 3: `procesar_followups_pendientes()`**
- Query all `FollowUpSchedule` rows where `status == "pending"` AND `scheduled_at <= datetime.utcnow()`.
- For each row:
  1. Check `obtener_estado(row.telefono)` from takeover.py — if "humano", SKIP (don't send, don't cancel — vendor may close)
  2. Send `FOLLOWUP_MESSAGE` via `proveedor.enviar_mensaje(row.telefono + "@s.whatsapp.net", FOLLOWUP_MESSAGE)`
  3. Set `row.status = "sent"`, `row.updated_at = utcnow()`
  4. Log success
- Return dict with counts: `{"processed": N, "sent": M, "skipped_humano": K}`
- Use lazy imports for `obtener_proveedor` and `obtener_estado` to avoid circular imports (same pattern as tools.py).
- Wrap each individual follow-up in try/except — one failure should not block others.
