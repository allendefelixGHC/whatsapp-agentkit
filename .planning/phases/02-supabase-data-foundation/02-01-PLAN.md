<task type="auto">
  <name>Task 2: Deep scraper with listing + detail page extraction</name>
  <files>agent/scraper.py</files>
  <action>
Create `agent/scraper.py` — a standalone scraper module that fetches Bertero listing pages + detail pages and persists to Supabase.

Key design decisions:
- Reuse the URL patterns and regex approach from existing `_parsear_listado` in tools.py, but output dicts matching the Supabase schema (not the display format).
- Detail page parsing uses the verified HTML structure from RESEARCH.md: `#lista_informacion_basica` for ambientes/dormitorios/banos/antiguedad/expensas, `#lista_superficies` for sup_cubierta/sup_total.
- Detail page price is AUTHORITATIVE (DATA-05): always use the price from the detail page heading, not the listing page price.
- Add `asyncio.sleep(0.5)` between detail page requests to avoid rate limiting.

Implementation:

```python
import re, asyncio, logging, httpx
from datetime import datetime, timezone
from agent.supabase_client import upsert_propiedades, marcar_removidas

BASE_URL = "https://www.inmobiliariabertero.com.ar"
logger = logging.getLogger("agentkit")
```

Functions to create:

1. `async def scrape_and_persist() -> dict`: Main entry point. Returns `{"total": N, "nuevas": M, "removidas": R}`.
   - Call `_scrape_listado_todas()` to get all property stubs (id, link, tipo, operacion, zona, direccion, listing_precio).
   - For each stub, call `_scrape_detalle(link)` to get full data. Use `asyncio.sleep(0.5)` between requests.
   - Build list of dicts matching Supabase schema. Set `scraped_at` to `datetime.now(timezone.utc).isoformat()`.
   - Call `upsert_propiedades(batch)` with all properties.
   - Call `marcar_removidas([p["propiedad_id"] for p in batch])` to clean up properties no longer on the website.
   - Return stats dict.

2. `async def _scrape_listado_todas() -> list[dict]`: Fetch pages 1-4 from `/Propiedades?p=N`. Parse each page with `_parsear_listado_raw(html)`. Return combined list of stubs.

3. `def _parsear_listado_raw(html: str) -> list[dict]`: Same logic as existing `_parsear_listado` in tools.py — find `href="/p/(\d+)-([^"]+)"` links, extract tipo/operacion/zona/direccion from slug, extract listing precio and superficie from HTML block. Return list of dicts with keys: propiedad_id, link, tipo, operacion, zona, direccion, precio (listing), precio_num (listing), superficie.

4. `async def _scrape_detalle(link: str) -> dict`: Fetch `BASE_URL + link`. Parse with `_parsear_detalle_campos(html)`. Return dict of detail fields.

5. `def _parsear_detalle_campos(html: str) -> dict`: Extract structured fields from detail page:
   - **Price (AUTHORITATIVE)**: `re.search(r'(USD|U\$S)\s*([\d.,]+)', html)` — this overrides listing page price.
   - **Info basica**: Parse `#lista_informacion_basica` `<li>` items. Extract ambientes (int), dormitorios (int), banos (int), antiguedad (str), expensas (str). Use `li.partition(':')` pattern from RESEARCH.md.
   - **Superficies**: Parse `#lista_superficies` `<li>` items. Extract sup_cubierta and sup_total (the key names are "cubierta" and "total construido").
   - **Description**: `re.search(r'class="[^"]*description[^"]*"[^>]*>(.*?)</div>', html, re.DOTALL)` — strip HTML tags, limit to 1000 chars.
   - Return dict with: precio, precio_num, ambientes, dormitorios, banos, sup_cubierta, sup_total, antiguedad, expensas, descripcion.

6. `def _safe_int(value: str, default: int = 0) -> int`: Helper to parse first digits from a string. E.g. "2" -> 2, "36 Años" -> 36.

IMPORTANT: When merging listing stub + detail data, the detail page precio/precio_num OVERRIDE the listing page values. If detail page has no price (rare), fall back to listing page price.
